Madoko

O Madoko é um processador Markdown escrito em JavaScript extremamente rápido usado para criar documentos escolares e industriais.

Visão geral

O Madoko é um processador Markdown bem simples de usar com opções para exportar o seu documento nos formatos PDF e HTML.

Apesar da simplicidade da ferramenta, você pode criar documentos complexos em Markdown e obter resultados impressionantes na conversão em HTML e PDF.

O nome Madoko vem de “Markdown in Koka”, já que a ferramenta foi desenvolvida com o intuito de demonstrar a linguagem de programação funcional, Koka.

O Madoko possui integração com Dropbox, Github e OneDrive, fornecendo assim, facilidade para o usuário manter seus documentos Markdown em qualquer dispositivo.

O programa detecta qualquer alteração nos arquivos e os sincroniza automaticamente na nuvem.

Você pode executar o Madoko localmente ou remotamente através do site madoko.net (o site no momento está com um problema de privacidade, por isso não vou colocar o link aqui).

Usando a opção online você aproveita os recursos de armazenamento e colaboração.

Em se tratando de compatibilidade, o Madoko está em outro nível, a ferramenta implementa extensões como GFM (GitHub Flavored Markdown), Markdown Extra , multi-markdown  e até o pandoc .

NOTA: O Madoko considera tabulação como 4 espaços, configure seu editor de texto para converter as TABs em 4 espaços para evitar problemas de formatação ao usar a processador.

Enquanto o site oficial do Madoko apresenta problemas de privacidade no certificado HTTPS, você pode visitar a página do projeto no GitHub em https://github.com/koka-lang/madoko .

Suporte ao Markdown no Madoko

A Tabela 1.1 apresenta os elementos Markdown compatíveis com o Madoko.

Elemento Suporte Obs
Títulos Sim
Parágrafos Sim
Quebras de linha Sim
Negrito Sim
Itálico Sim
Bloco de citação Sim
Listas ordenadas Sim
Listas não ordenadas Sim
Código Sim
Linhas horizontais Sim
Links Sim
Imagens Parcial Você pode inserir as imagens diretamente no seu documento, elas serão carregadas e vinculadas a um subdiretório dentro da pasta do arquivo. Imagens com mais de 1 Mega Byte não são permitidas no servidor do Madoko.
Tabelas Sim O Madoko possui compatibilidade estendida da sintaxe da tabela .
Blocos de código cercado Sim Há compatibilidade com o realce do conteúdo do bloco. O Madoko usa o Monarch internamente para a tarefa.
Notas de rodapé Sim
Id de títulos Não
Listas de definições Sim
Riscado Sim
Listas de tarefas Sim
Emoji (copiar e colar) Sim
Emoji (códigos de acesso) Não
Link automático de URL Sim
Desativar o link automático de URL Não
HTML Parcial Se precisar de blocos personalizados, consulte a Referência do Madoko (em inglês) .
